# swaydots

[Sway](https://github.com/swaywm/sway) dotfiles for **Fedora** (and similar setups): tiling WM, **Waybar**, **Fuzzel** launcher, **Kitty** terminal, **wlogout**, wallpapers, screenshots, and a **unified theme switcher** with **18** dark and light palettes (Catppuccin variants, Nord, Gruvbox, Dracula, Tokyo Night, Solarized, One, Everforest, Rosé Pine, Rainbow, and more).

Repository: [github.com/toppzi/swaydots](https://github.com/toppzi/swaydots)

## Contents

| Path | Purpose |
|------|---------|
| `sway/` | Main config, **`config.d/`** (e.g. **`20-waybar.conf`** autostarts Waybar under Sway), wallpaper scripts + GTK picker, screenshots, **`theme-switch.sh`**, **`themes/`** (palettes + CSS/INI templates) |
| `hypr/` | **`hyprland.conf`** starter (Waybar **`exec-once`**, polkit, kitty/fuzzel binds) — installed only when Hyprland is present or selected |
| `waybar/` | Bar layout (`config.jsonc`); **`style.css`** is regenerated by the theme switcher |
| `wlogout/` | Layout and style (icons from the `wlogout` package) |
| `kitty/` | `kitty.conf` + **`themes/*.conf`**; active colours are **`themes/theme-active.conf`** |
| `fuzzel/` | Starter `fuzzel.ini` (overwritten when you change theme) |
| `install.sh` / `pack.sh` | Install into `~/.config` · pack from live config back into this tree |

Waybar can open **btop** in Kitty (`kitty -e btop`); the installer pulls in **btop** when using `dnf`.

## Install

```bash
cd /path/to/sway-dotfiles
chmod +x install.sh pack.sh
./install.sh
```

What **`./install.sh`** does:

1. Shows a short banner (ANSI colours on a TTY; set **`NO_COLOR=1`** to disable).
2. Prompts for a **wallpaper directory** (default **`~/Pictures/wallpapers`**) unless you pass **`--wallpaper-dir`**; that path is used later for **`environment.d`** and **`~/Pictures/wallpapers`** is created if needed.
3. **Keyboard (XKB):** interactive menu shows a **number → layout** map (e.g. **1=us**, **2=se**, **3=gb** UK, **4=no** Norwegian, …) unless you pass **`--keyboard-layout`**. Writes **`~/.config/sway/config.d/10-xkb-layout.conf`** and, when Hyprland dotfiles are installed, sets **`kb_layout`** in **`~/.config/hypr/hyprland.conf`**. Non-interactive installs default to **us** unless **`--keyboard-layout`** is set.
4. **Compositor:** if neither **Sway** nor **Hyprland** is installed (RPM), asks which to **`dnf install`** with an explicit map (**1 = sway**, **2 = hyprland**). The script **verifies** the compositor RPM after `dnf` and **stops with an error** if it is still missing (earlier versions hid `dnf` failures). Use **`--compositor sway|hyprland`** when non-interactive. Shared stack either way: Waybar, wlogout, Kitty, Fuzzel, grim/slurp, etc.
   - **Sway:** **`sway`**, **`swaylock`**, **`swayidle`**, **`swaybg`**, **`xdg-desktop-portal-wlr`** (wlroots portal), **`xdg-desktop-portal-gtk`**, **`mate-polkit`** (GTK polkit agent; **`polkit-gnome`** is not shipped on Fedora 43+), **`qt6-qtwayland`**, **`wireplumber`**, **`sway-systemd`** (Fedora session integration). If Sway was already installed, those extras are still ensured (without reinstalling **`sway`** itself).
   - **Hyprland:** **`hyprland`**, **`xdg-desktop-portal-hyprland`**, **`xdg-desktop-portal-gtk`**, **`mate-polkit`**, **`qt6-qtwayland`**, **`wireplumber`**. If Hyprland was already installed, session packages are still ensured.
5. If no display manager package (**sddm**, **lightdm**, **gdm**, **ly**) is installed, asks which to install with a **number → DM** map (**1=sddm** … **4=ly**, **q** skip), or use **`--display-manager`**, **`--skip-display-manager`**, **`--no-packages`**. **Ly** is often **not** in Fedora’s default repos: the installer enables the **[fnux/ly](https://copr.fedorainfracloud.org/coprs/fnux/ly/)** COPR if needed, then disables **`getty@tty2`** so Ly can use **tty2** (reboot after install).
6. On Fedora (when **`dnf`** is available and you do not pass **`--no-packages`**):
   - Runs **`sudo dnf install`** for **`dnf-plugins-core`**, the compositor package(s) above (if needed), Waybar, wlogout, Kitty, Fuzzel, **btop**, grim/slurp, wl-clipboard, **gettext** (**`envsubst`** for themes), Python/GTK for the wallpaper picker, and fonts.
   - Installs **pavucontrol** if it is missing.
   - Enables the **[linuxgamerlife/lgl-system-loadout](https://copr.fedorainfracloud.org/coprs/linuxgamerlife/lgl-system-loadout/)** COPR and installs **`lgl-system-loadout`** ([LGL System Loadout](https://github.com/linuxgamerlife/lgl-system-loadout) — see **Thanks** below).
   - Installs the chosen display manager (if any) and enables its systemd service.
7. Writes **`WALLPAPER_DIR`** to **`~/.config/environment.d/99-sway-dotfiles-wallpaper.conf`**.
8. Backs up existing `~/.config/{sway,waybar,wlogout,kitty,fuzzel}` (and **`hypr`** if Hyprland is in use) with a timestamp if present, then copies those folders from the repo, applies the chosen **keyboard** layout to Sway and Hyprland configs, and continues.
9. Makes shell scripts executable and, if **`envsubst`** and **`~/.config/sway/themes/palettes/`** exist, runs **`theme-switch.sh catppuccin --no-reload`** once so Waybar, Fuzzel, Kitty, GTK, and **Sway** border colours line up (Sway-specific pieces are skipped at runtime under Hyprland).

**Hyprland:** when Hyprland is installed or selected, the installer copies **`hypr/hyprland.conf`**, which **`exec-once`s Waybar**, dbus activation env, and **`mate-polkit`**’s agent (see comments for **`hyprpolkitagent`** as a COPR alternative). Sway-only installs skip **`~/.config/hypr`**. Extend **`hyprland.conf`** with monitors from **`hyprctl monitors`** and binds from the [Hyprland wiki](https://wiki.hyprland.org/Configuring/).

**Hyprland on Fedora:** the **`hyprland`** RPM is not always in the default repos on a minimal install. This installer, if **`dnf`** fails while you chose Hyprland, enables the **[solopasha/hyprland](https://copr.fedorainfracloud.org/coprs/solopasha/hyprland/)** COPR on Fedora and retries (same idea as the community walkthrough [Tutorial: Fedora 43 — Install Hyprland from scratch](https://discussion.fedoraproject.org/t/tutorial-fedora-43-install-hyprland-from-scratch/168386)). If install still fails, that thread and **`dnf`** output are the best next steps.

**After install:** log out and back in (or reboot) so **`environment.d`** applies and **`wallpaper.sh` / `wallpaper-picker.py`** see **`WALLPAPER_DIR`**. If a display manager was installed or switched, reboot (or start that service) so graphical login uses it.

### Installer flags

| Flag | Meaning |
|------|---------|
| `--dry-run` | Print actions only |
| `--no-packages` | Skip `dnf`; you install packages yourself |
| `--wallpaper-dir PATH` | Set wallpaper path without prompting (`~` expanded) |
| `--display-manager NAME` | When no DM is installed: install **NAME** (`sddm`, `lightdm`, `gdm`, or `ly`) without prompting |
| `--skip-display-manager` | Do not check, prompt, or install a display manager |
| `--compositor NAME` | When neither Sway nor Hyprland is installed: install **NAME** (`sway` or `hyprland`) without prompting |
| `--keyboard-layout CODE` | XKB layout for Sway and Hyprland (e.g. `us`, `se`, `gb`, `no`; **`uk`** is accepted as **`gb`**) |
| `-h` / `--help` | Usage |

## Theme switcher

**18 themes** — run **`~/.config/sway/theme-switch.sh list`** for exact ids. Overview:

| Dark | Light |
|------|-------|
| catppuccin (Mocha), catppuccin-frappe, catppuccin-macchiato | **catppuccin-latte** |
| dracula, everforest-dark, gruvbox, nord, one-dark, rose-pine | **everforest-light**, **one-light**, **rose-pine-dawn** |
| solarized-dark, tokyo-night | **solarized-light**, **tokyo-night-day** |
| rainbow (multi-accent) | — |

Each run updates:

- **`~/.config/sway/config.d/40-theme.conf`** — Sway window borders  
- **`~/.config/waybar/style.css`**  
- **`~/.config/kitty/themes/theme-active.conf`**  
- **`~/.config/fuzzel/fuzzel.ini`**  
- **`~/.config/gtk-3.0/settings.ini`** and **`gtk-4.0/settings.ini`** — **Thunar** and other GTK apps  

| Action | How |
|--------|-----|
| Pick in a menu | **Super+Shift+t** (`Mod+Shift+t`) |
| List names | `~/.config/sway/theme-switch.sh list` |
| Apply by name | `~/.config/sway/theme-switch.sh nord` (example) |

After a switch, Sway reloads and Waybar/Kitty get refresh signals. **Restart Thunar** if GTK chrome does not update.

**GTK:** Names in **`sway/themes/palettes/*.env`** (`GTK_THEME=…`) must match themes installed on your system, or GTK will fall back. Light palettes use **`Adwaita`**; most dark ones use **`Adwaita-dark`**. Adjust **`GTK_THEME`** per `.env` if you use Nordic, Dracula GTK, etc.

Palette keys and templates are documented in **`sway/themes/README.md`**.

### Upgrading after `git pull`

The switcher needs the **whole** tree **`~/.config/sway/themes/`** (`palettes/` and **`tpl/`**), not only **`theme-switch.sh`**. If **Super+Shift+t** stops working after an update, run **`./install.sh`** again from the repo or copy manually:

```bash
cp -a /path/to/sway-dotfiles/sway/themes ~/.config/sway/
cp -a /path/to/sway-dotfiles/sway/theme-switch.sh ~/.config/sway/
chmod +x ~/.config/sway/theme-switch.sh
```

Then **Super+Shift+c** to reload Sway.

### Troubleshooting themes

- **Depends on:** **`fuzzel`**, **`gettext`** / **`envsubst`**, executable **`theme-switch.sh`**, and **`~/.config/sway/themes/`** as above.
- The script prepends a standard **`PATH`** (`/usr/bin`, etc.) because Sway **`exec`** often uses a minimal environment.
- The keybinding runs **`/bin/bash $HOME/.config/sway/theme-switch.sh`**.
- Run **`~/.config/sway/theme-switch.sh list`** in a terminal; failures usually print to stderr and may show a **notify-send** alert.

## After a fresh install

Edit **`~/.config/sway/config`** for your hardware and habits:

- **`output …`** for monitors — use **`swaymsg -t get_outputs`**
- **`$browser`**, **`$filemanager`**, workspace → output if needed
- Optional **autotiling:** `pip install --user autotiling` and ensure **`~/.local/bin`** is on **`PATH`** in the Sway session

Reload config: **Super+Shift+c** (default **`$mod`** is Super).

## Pack (reverse sync)

To copy your live configs back into this repo before committing:

```bash
./pack.sh
```

This rsyncs **`sway`**, **`waybar`**, **`wlogout`**, and (if they exist under **`~/.config`**) **`kitty`** and **`fuzzel`**. It does **not** create **`kitty/`** or **`fuzzel/`** in the repo until those dirs exist on your machine.
